定时执行全量任务

全量任务：synchronize方法，如果已经有indexpath_new的文件夹，先删除，之后生成索引文件夹：indexpath_new，生成结束之后，
调用锁定消息将是否可用置为false。判断当前的检索线程数，如果为0则删除indexpath，并将new改名

定时任务：定时调度全量任务即可。提供启动、停止、时间调度的接口

检索：检索之前先判断索引是否可用：调用锁定消息获得状态，若状态为可用则检索数量+1，用完了检索数量-1。不可用则等待5s，超过5s则报错。

锁定消息：存储检索数量和是否可用两个变量，检索数量为正在检索的线程数，是否可用由全量任务调用。
